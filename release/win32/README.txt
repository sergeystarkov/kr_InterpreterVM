Разработчик: Старков С.А.
Назначение: Курсовое проектирование по предмету "Системное программное обеспечение"
Название: Интерпретатор виртуальной машины.

Краткая характеристика архитектуры виртуальной машины.

Пользователю доступны для программирования 2 регистра данных размерностью 4 байта - сумматор и внутренний регистр.
Адресный регистр для промежуточного храниения адреса размерностью 2 байта.
Объём оперативной памяти ограничен возможностью адресации в 16 бит.
Система команд процессора состоит из 23 команд + 23 команды для альтернативной адресации операнда команды.
Перечень команд процессора можно найти в документе "Перечень команд процессора.txt" в папке ./help

Типы адресации:
1) абсолютная, означает что константа в операнде и есть адрес.
2) относительная (для обозначения используется вторая линейка из 23 команд процессора), означает, что константа в операнде - это отступ от адресного регистра (адрес = адресный регистр + константа в операнде).

Все программы должны соблюдать синтаксис:

хх уу
0

,где 	xx - команда процессора;
	уу - константа (адрес).

Завершаться программа должна по коду операции 0

Во время загрузки и выполнения программы могут возникать ошибки. Ошибка завершает выполнение программы и возвращает её код.

Коды ошибок и их расшифровка:

0 - нет ошибки, программа корректно завершилась
1 - программа в данный момент штатно выполняется
2 - ошибка при загрузке программы, возможно ошибка в синтаксисе
3 - неизвестная команда
4 - адресный регистр вышел за пределы адресуемой памяти.

Краткая инструкция по запуску и работе интерпретатора виртуальной машины.

1) Запукстите программу CW_InterpreterVM.exe

2) Нажмите на кнопку открыть и выберите файл программы из папки ./program или вручную введите программу в левое поле.

3) При необходимости отредактируйте и сохраните файл нажатием на кнопку "Сохранить" или "Сохранить как"

4) Нажмите на кнопку "Запустить интерпретатор", при этом программа начнёт выполняться.

5) В зависимости от написанной программы, могут высвечиваться сообщения или запросы на ввод числа.

6) В правом поле можно наблюдать на отладочные сообщения программы, где описаны, какие действия она сейчас производит.

7) По завершении работы программы можно закрыть приложение либо запустить программу повторно.
